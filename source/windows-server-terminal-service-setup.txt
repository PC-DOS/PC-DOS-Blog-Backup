Server 2019默认远程桌面连接数是2个用户，如果多余两个用户进行远程桌面连接时，系统就会提示超过连接数，可以通过添加远程桌面授权解决：

+ 添加远程桌面授权服务

 启动服务器管理，点击 [管理] > [添加角色和功能] 菜单项，打开[添加角色和功能向导]窗口，在 [服务器角色] 选单中，勾选 [远程桌面服务] 下的“Remote Desktop Session Host”和“远程桌面授权”，并安装角色。

 您可能需要重新启动目标服务器。

 以上配置完成后即可使用多于2用户同时登录，但使用期限为120天。

+ 添加远程桌面授权许可

 添加之前将时间调至未来的一个时间，用来增加使用期限。打开 [服务器管理器] > [远程桌面服务] > [服务器]。右键单击目标服务器，选择 [RD 授权管理器]，在弹出的控制台窗口中可以看到服务器目前为未激活状态。

 右键单击目标服务器，点击 [激活服务器]，打开 [服务器激活向导]。[连接方法] 选择 [Web浏览器]，系统将自动打开远程桌面授权网站。

 选择 [启用许可证服务器]，输入 [服务器激活向导] 中显示的产品ID，其余信息可自由填写。随后，即可获取许可证ID并将其粘贴到 [服务器激活向导] 中。至此，即可激活许可证服务器。保持勾选 [立即启动许可证安装向导] 并点击 [完成] 按钮，结束 [服务器激活向导] 并打开 [许可证安装向导]。

 在 [许可证安装向导] 中，[连接方法] 选择 [Web浏览器]，系统将自动打开远程桌面授权网站。选择 [安装客户端访问许可证] 并点击 [下一步]。在开启的页面中，[许可证程序] 选择 [企业协议]，[许可证服务器] 输入 [许可证安装向导] 中显示的许可证服务器ID，其他信息可以随意填写。点击 [下一步]，[产品类型] 根据您的Windows Server版本选择对应的 [每用户访问许可]，[协议号码] 可以填写{{6565792}}、{{4954438}}、{{6879321}}[[footnote]][*https://cloud.tencent.com/developer/article/1598570 有实践认为]该协议号（{{6879321}}）不可用。[[/footnote]]或者{{5296992}}，数量可以任意填写。点击 [下一步]，即可获取许可证密钥包ID。随后，在[许可证安装向导] 中输入许可证密钥包ID，即可完成激活。

+ 配置远程桌面授权模式

 按 [Windows徽标键]+[R] 组合键，打开 [运行] 对话框，输入“{{gpedit.msc}}”并按回车键，打开 [本地组策略编辑器]。

 在左侧窗格，依次展开 [计算机配置] > [管理模板] > [Windows组件] > [远程桌面服务] > [远程桌面会话主机] > [授权]。

 在右侧窗格双击 [使用指定的远程桌面许可证服务器]，在弹出的对话框中选择 [已启用]，在下方 [要使用的许可证服务器] 文本框输入本地计算机的计算机名和/或IP地址（使用半角逗号分隔），单击 [确定]按钮关闭对话框。

 在右侧窗格双击 [设置远程桌面授权模式]，在弹出的对话框中选择 [已启用]，在下方 [指定RD会话主机服务器的授权模式] 下拉列表框选择 [按用户]，单击 [确定]按钮关闭对话框。

 至此，整个安装过程已全部完成。

+ 疑难排解

 如果远程桌面授权服务器提供的许可证过期，则将无法远程登陆，解决方法如下：

 使用{{mstsc /admin /v:IP_ADDRESS}}来强制登录服务器（注意只能管理员身份登录）。

 调整日期为未来的时间（获得更长的时间，不更改可再次获得120天授权）。

 删除注册表项{{HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\RCM\GracePeriod}}。
