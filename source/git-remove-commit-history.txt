该方法可用于删除错误的提交记录（例如包含私密内容的提交记录、使用了错误的用户名或电子邮件地址的提交记录等）。开始前，请存档所有需要回退的提交中的代码、数据和文件，以防万一。

 首先，在本地仓库执行{{git reset --hard}}命令，强制回退：

[[code]]
 git reset --hard HEAD~X
 [[/code]]

其中，“X”为需要回退的提交（Commit）计数。例如，若您需要回退到上一个提交，则X=1；若您需要回退到上上个提交，则X=2……以此类推。

 此时，本地的提交记录完成回退。在进行远程提交记录的回退前，请确保目标仓库（分支）的分支保护功能已关闭，并允许强制推送。

 随后，进行git强制推送，向远程进行同步：

[[code]]
 git push -f origin
 [[/code]]

此时，已完成提交记录的回退。
